# 画布结构与图层治理

本页记录从预期稿 `TOtyDTf2` 和问题截图中沉淀出的画布结构硬规则。目标是避免 AI 生成稿出现外层自动布局、背景层丢失、tabbar 贴底、图层树混乱等问题。若后续可读取 MasterGo 节点属性，以节点真实属性为准；读取不到节点属性时，按本页观察规则执行。

## 一级界面画布结构

1. 单个设计稿画板必须是普通 `Frame`，尺寸为 `360 x 780`。
2. `360 x 780` 这一层只做画板容器，不要设置自动布局。
3. 画板内部的 `内容区`、`卡片组`、`top`、列表、网格行等语义容器应使用 flex / 自动布局承载组件排布和自适应。
4. 一级界面底色为 `#FFFFFF`。
5. 一级界面必须放置切片 `一级界面顶部渐变`。
6. `一级界面顶部渐变` 必须顶到最上方，即从 Frame 的 `y=0` 开始。
7. `一级界面顶部渐变` 位于状态栏和内容层下方，不应压住可交互组件。
8. 状态栏和顶部导航栏之间的间距为 `0`。
9. tabbar 悬浮在界面上，距离底部安全距离 `16dp`。

推荐层级顺序：

```text
一级界面 Frame 360x780 #FFFFFF
├─ 背景层
│  └─ 一级界面顶部渐变
├─ 状态栏
├─ 顶部导航栏
├─ 内容区（flex / 自动布局）
│  ├─ top（flex / 自动布局）
│  ├─ 卡片组（flex / 自动布局）
│  └─ 键盘组件（仅键盘场景）
└─ 悬浮层
   └─ tabbar（bottom safe area = 16dp）
```

## 顶部结构规则

- 状态栏紧贴 Frame 顶部。
- 顶部导航栏紧贴状态栏，二者之间不能插入空白间距。
- 顶部渐变切片从 `y=0` 开始铺在背景层，覆盖状态栏和导航栏背后的视觉区域。
- 如果使用 `Navbar 导航` 组件，组件上方不再额外加 spacer。

## tabbar 规则

- tabbar 是悬浮层，不参与内容区正常流式排版。
- tabbar 底部到一级界面 Frame 底部的安全距离为 `16dp`。
- 内容区需要预留 tabbar 遮挡空间，不能让卡片、键盘或输入条被 tabbar 压住。
- tabbar 只能有一个当前选中项。

## 图层治理规则

AI 生成稿图层混乱的主要原因：

1. 把页面根节点做成自动布局，导致所有绝对层、悬浮层、背景切片都被挤进同一个流式结构。
2. 把组件实例、标注层、临时容器、素材层混在同一层级。
3. 为了对齐截图给大量子组件写死 `x/y/w/h`，后续组件自适应被破坏。
4. 对相同语义区域重复建 frame，例如卡片外层、卡片内容、卡片背景各自独立散落。
5. 没有固定的命名和分组规则，导致 c2d 后图层树无法阅读。

解决方式：

- 单个 `360 x 780` 画板只做普通 Frame，不做自动布局。
- 画板内部组件必须优先放入语义化 flex / 自动布局容器，例如 `内容区`、`卡片组`、`top`，让组件跟随父容器自适应。
- 背景层、状态栏、导航栏、内容区、悬浮层必须分组清楚。
- 标注、红框、箭头、说明文字不得进入交付界面 Frame；需要说明时放在独立说明区。
- 组件实例保持为组件实例，不拆散为内部零件。
- 临时 frame 只能用于组件库缺失的容器骨架，并且必须命名为 `临时容器-用途`。
- 同一语义只保留一个父容器；不要让卡片、标题、图片、标签散落在页面根层。

## 命名规则

推荐命名：

| 层级 | 命名 |
| --- | --- |
| 页面根 | `一级界面-页面名` |
| 背景层 | `背景层` |
| 顶部渐变 | `一级界面顶部渐变` |
| 状态栏 | `状态栏` |
| 顶部导航 | `顶部导航栏` |
| 内容区 | `内容区` |
| 分类区 | `分类导航区` |
| 卡片组 | `卡片组-用途` |
| 悬浮层 | `悬浮层` |
| tabbar | `tabbar` |

## 交付前自检

- [ ] 单个设计稿画板是普通 Frame，尺寸 `360 x 780`，不是自动布局。
- [ ] 画板内部的 `内容区`、`卡片组`、`top` 等语义容器使用 flex / 自动布局。
- [ ] 一级界面 Frame 背景为 `#FFFFFF`。
- [ ] `一级界面顶部渐变` 存在，并且顶到 `y=0`。
- [ ] 状态栏与顶部导航栏间距为 `0`。
- [ ] nav 以外内容区左右边距为 `16`。
- [ ] 卡片组间距为 `8`。
- [ ] tabbar 在悬浮层，底部安全距离为 `16dp`。
- [ ] 类 tab 选择组只有一个选中态。
- [ ] 组件实例未被拆散，键盘使用 `键盘组件`。
- [ ] 标注、红框、箭头未混入交付 Frame。
- [ ] 图层按背景层 / 状态栏 / 顶部导航栏 / 内容区 / 悬浮层分组。
