# 画布与图层规则

目标：保证生成稿像设计稿，而不是一堆散乱图层。手机画板是普通 Frame；组件排布靠画板内部语义容器的 flex / 自动布局完成。

## 画板硬规则

1. 单个设计稿画板尺寸固定为 `360 x 780`。
2. `360 x 780` 这一层必须是普通 Frame / 画板，不是自动布局。
3. 画板背景为 `#FFFFFF`。
4. 画板内部才使用 flex / 自动布局，例如 `内容区`、`top`、`卡片组`、列表、网格行、bottom sheet、工具栏。
5. 流程箭头、红框、点击标注、说明文字放在手机画板外，不混入交付 Frame。

## 一级界面结构

```text
一级界面-页面名 Frame 360x780 #FFFFFF
├─ 背景层
│  └─ 一级界面顶部渐变（y=0）
├─ 状态栏
├─ 顶部导航栏
├─ 内容区（flex / 自动布局）
│  ├─ top（可选，承载 tab / 筛选 / 二级导航）
│  ├─ 卡片组-用途（gap=8）
│  └─ 键盘组件（仅键盘场景）
└─ 悬浮层
   ├─ tabbar（bottom safe area=16）
   └─ bottom sheet / toast / 弹窗
```

## 间距和自适应

- nav 以外的内容区左右边距为 `16`。
- 卡片组之间的间距为 `8`。
- 状态栏和顶部导航栏之间间距为 `0`。
- `一级界面顶部渐变` 必须顶到 `y=0`，放在背景层，不压住可交互组件。
- `tabbar` 悬浮在画板上，距离底部安全距离 `16dp`。
- 组件实例不要锁死内部宽高；优先 `self-stretch` 或由父容器控制。
- 只有画板尺寸、内容边距、卡片间距、tabbar 安全距离等规范值需要明确。

## 图层治理

1. 根 Frame 下按 `背景层 / 状态栏 / 顶部导航栏 / 内容区 / 悬浮层` 分组。
2. 同一语义区域只保留一个父容器。
3. 组件实例保持为组件实例，不拆散成零件。
4. 临时 frame 只能用于组件库缺失的容器骨架，并命名为 `临时容器-用途`。
5. 不为了贴合截图给大量子节点写死 `x/y/w/h`。
6. 需要画多个手机界面时，每个手机界面都是独立 `360 x 780` Frame，流程箭头和页面标题在 Frame 外层。

## 命名规则

| 层级 | 命名 |
| --- | --- |
| 页面根 | `一级界面-页面名` / `二级界面-页面名` |
| 背景层 | `背景层` |
| 顶部渐变 | `一级界面顶部渐变` |
| 状态栏 | `状态栏` |
| 顶部导航 | `顶部导航栏` |
| 内容区 | `内容区` |
| 顶部控制区 | `top-用途` |
| 卡片组 | `卡片组-用途` |
| 悬浮层 | `悬浮层` |
| 临时容器 | `临时容器-用途` |
| 冲突说明 | `设计冲突说明` |

## 交付前自检

- [ ] 每个手机画板是普通 Frame，尺寸 `360 x 780`。
- [ ] Frame 背景为 `#FFFFFF`。
- [ ] 一级界面包含 `一级界面顶部渐变`，且在 `y=0`。
- [ ] 状态栏与顶部导航栏间距为 `0`。
- [ ] nav 外内容区左右边距为 `16`。
- [ ] 卡片组 gap 为 `8`。
- [ ] tabbar 在悬浮层，底部安全距离为 `16dp`。
- [ ] 类 tab 选择组只有一个选中态。
- [ ] 键盘场景使用 `键盘组件`。
- [ ] 标注、红框、箭头未混入手机 Frame。
