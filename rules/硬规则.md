# 硬规则

## 设计流程

1. 生成前必须先读交互截图，完成 OCR/视觉解读。
2. 生成前必须读本 skill 文档和本目录 references。
3. 生成前必须读 MasterGo 组件库 `https://mastergo.iflytek.com/goto/TOAnEBPS` 对应的本地快照。
4. 设计时先从组件库抽取候选组件，能用组件就用组件。
5. 不要动不动自创图层；临时 frame 只用于组件缺失的容器骨架。
6. 提交前必须运行 `scripts/check_mastergo_html.py`，有 ERROR 时不能提交。

## 画板和布局

- `360 x 780` 是单个设计稿画板容器尺寸，不是内容区尺寸。
- 单个设计稿画板必须是普通 Frame，不能是自动布局。
- 画板内部组件和语义容器应使用 flex / 自动布局组织，例如 `内容区`、`卡片组`、`top`、列表、网格行、弹层内容和工具栏。
- 一级界面背景色为 `#FFFFFF`。
- 一级界面必须包含 `一级界面顶部渐变`，且该切片顶到 Frame 最上方 `y=0`。
- 状态栏和顶部导航栏间距为 `0`。
- nav 可以贴边或遵循自身组件规范；nav 以外的内容容器左右边距为 `16`。
- 卡片组之间使用 `8` 间距。
- tabbar 是悬浮层，距离底部安全距离 `16dp`。
- 组件实例尽量使用自适应布局：画板内部的 flex 容器负责 `16` 边距和 `8` 卡片组间距；组件自身宽度优先 `self-stretch` 或由父容器控制。
- 除画板基准、内容边距、卡片组间距、明确的组件规范尺寸外，不要给组件内部写死固定宽高。

## 交互状态

- 类 tab 的选择组只能有一个选中态。
- 同一组内只能存在一个 `checked` / `selected` / `选中`。
- 其余必须为 `unchecked` / `unselected` / `未选中`。
- 箭头、红框、说明文字、点击标注不能混入交付界面 Frame；流程图标注放在画板外层。

## 禁止项

- 禁止把 `1:1 图片素材`、`3 纵列图片素材`、`placeholder_pic` 当作业务组件调用。
- 禁止用临时 frame 代替 `键盘组件`。
- 禁止组件实例拆散成零件。
- 禁止没有依据地改写组件 name、props、slots、instance_swap。
- 禁止没有 OCR/截图解读就直接生成界面。
- 禁止没有跑 hook 就提交到 MasterGo。

## 冲突处理

当 design spec、组件说明、变量、截图之间冲突：

1. 不要自行静默取舍。
2. 在设计稿画板外新增 `设计冲突说明`。
3. 写清冲突点、来源、临时处理建议。
4. 涉及核心业务组件、素材替代组件、变量冲突时，停下来询问用户。
