# 组件调用

## 组件选择优先级

1. 能表达完整业务块：选择以 `****组件` 命名的复合组件。
2. 业务卡片优先选 `封面卡片组件`、`图标卡片组件`、`点击卡片组件`、`上传卡片组件`。
3. 键盘区域直接调用 keyboard/键盘 下的 `键盘组件`。
4. 表单、列表、导航、弹窗、反馈使用对应基础组件：`button`、`input`、`radio`、`checkbox`、`toggle`、`list_switch`、`list_enter`、`modal`、`toast`、`placeholder`。
5. 导航优先调用 `nav`；局部 tab 才调用 `nav_Level1_tab` / `nav_Level2_tab` / `nav_Level3_tab` / `nav_Level4_tab`。
6. 组件没有描述、cover 或可靠变体说明时，宁可用命名清楚的临时 frame 复刻骨架，也不要猜组件能力。

## 禁止直接调用的组件入口

除非用户明确要求还原素材本身，否则不要在 c2d 页面生成中调用：

- `1:1 图片素材`
- `3 纵列图片素材`
- `placeholder_pic`

除非正在编辑复合组件内部结构，否则不要优先调用：

- `卡片_封面`
- `卡片_内容区`
- `图标卡片组件-图标`
- `withicon`

## 常见组合

| 场景 | 父组件 | 子/伴随组件 | 规则 |
| --- | --- | --- | --- |
| 首页一级导航 | `nav` | `nav_Level1_tab` | 完整导航优先 `nav` |
| 封面商品卡 | `封面卡片组件` | `类别标签` / `玩法标签（素材营销、账号权益）` | c2d 调用父组件，不拆内部 |
| 图标分类卡 | `图标卡片组件` | `图标卡片组件-图标` | c2d 调用父组件 |
| 点击选择卡 | `点击卡片组件` | 无 | 表情位、素材位优先考虑 |
| 上传入口 | `上传卡片组件` | 无 | 上传、回显、选项状态 |
| 键盘展示 | `键盘组件` | 无 | 画键盘必须调用 |
| 列表选择项 | `list_switch` | `radio` / `checkbox` / `toggle` | 用 props 控制状态 |
| 进入型列表 | `list_enter` | 图片、描述、标记区 | 按 props 变体选择 |
| 抽屉输入 | 临时容器 | `drawer_header` / `drawer_input` / `input` / `button_group` | 当前库无完整 drawer 父组件 |
| 弹窗操作 | `modal` | `button` / `button_group` / `option` | 反馈组件直接调用 |

## 组件匹配和抽取

当用户给的是交互截图：

1. 先识别每个页面区域：导航、tab、卡片、素材网格、弹层、键盘、按钮、输入框、反馈。
2. 对每个区域列出候选组件。
3. 对比截图样式和组件库样式/逻辑：
   - 视觉结构相近，优先组件。
   - 交互逻辑相近，优先组件。
   - 只有文字或图片不同，仍优先组件。
   - 父子关系相近，优先父组件。
4. 先在画布中放置组件实例，或在流程板外建立 `组件取用区`，再组合页面。
5. 临时 frame 只用于组件缺失处，并命名 `临时容器-用途`。

## HTML 调用示例

```html
<ui-component
  data-name="keyboard"
  name="键盘组件"
  class="self-stretch"
  props='{"属性 1":"按键皮肤键盘 1"}'
></ui-component>
```

```html
<ui-component
  data-name="cover-card"
  name="封面卡片组件"
  props='{"属性 1":"皮肤","属性 2":"价格"}'
></ui-component>
```
