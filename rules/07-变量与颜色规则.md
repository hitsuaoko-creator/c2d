# 变量与颜色规则

变量来源只认 `.mastergo/library/local-196518374727340/新设计变量/variable.json`。变量名以 JSON 中 `name` 原文为准。

## 使用原则

1. 能匹配变量时，使用 `var(变量 name)`。
2. 不直接抄变量 `value`，除非变量库没有该类型。
3. 浅色界面使用 `浅色_index_1/*`，深色界面使用 `深色_index_1/*`，不要混用。
4. 当前变量只有颜色，没有字号、间距、圆角、阴影变量；这些属性按 MasterGo 页面生成规则和本 skill 的间距规则写字面值。
5. 一级界面底色规则优先级高：Frame 背景为 `#FFFFFF`。如需说明变量冲突，写入 `设计冲突说明`。

## 示例

```html
<div
  data-name="content-card"
  class="bg-[var(浅色_index_1/10 - 一级卡片色)] border-[1px] border-[var(浅色_index_1/19-描边色)]"
></div>
```

```html
<p
  data-name="secondary-text"
  class="text-[14px] leading-[20px] font-[400] text-[var(浅色_index_1/2 - 二级文本色)]"
>说明文本</p>
```

## 冲突处理

- 变量缺失：允许局部字面值，并在提交前报告里写明。
- 浅色有变量、深色没有对应变量：不要臆造深色变量。
- 组件视觉和变量文档冲突：以组件 JSON 事实和用户当前目标优先，并在 `设计冲突说明` 中记录。
