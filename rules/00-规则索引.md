# 输入法APP 设计规范引导

本目录是 `c2d` 的规则源。目标是让 AI 在生成 MasterGo 输入法 APP 设计稿时：先理解需求和截图，再读取组件库，最后用组件化方式生成设计稿。

组件库来源：

- MasterGo 文件：`https://mastergo.iflytek.com/file/196518374727340?file=196518374727340&page_id=1%3A6578`
- 文件名：`新设计变量`
- 本地快照：`.mastergo/library/local-196518374727340/新设计变量`
- 当前组件：`45` 个
- 当前变量：`35` 个

## 读取顺序

必读：

1. `01-需求引导.md`
2. `00-规则索引.md`
3. `02-组件使用规则.md`
4. `03-画布与图层规则.md`
5. `04-缺失冲突与校验规则.md`

按需读：

- 有 visual-insight 产物：读 `05-视觉产物解读规则.md`
- 需要确认组件名、props、slots、变量名：读 `06-组件与变量清单.md`
- 涉及颜色、明暗模式、变量冲突：读 `07-变量与颜色规则.md`

## 死规则

1. 生成前先读交互截图并输出 `交互截图解读`。
2. 有 visual-insight 产物时，先总结为 `本次视觉生成概要`，再生成设计。
3. 生成前必须刷新或读取 MasterGo 组件库快照。
4. 相近组件存在时，必须优先使用组件库组件。
5. 不使用 `1:1 图片素材`、`3 纵列图片素材`、`placeholder_pic` 生成业务 UI。
6. 画键盘必须使用 `键盘组件`。
7. `360 x 780` 手机设计稿外层必须是普通 Frame，不是自动布局。
8. Frame 内部的 `内容区`、`top`、`卡片组`、列表、弹层、工具栏使用 flex / 自动布局。
9. 一级界面背景为 `#FFFFFF`，`一级界面顶部渐变` 放在 `y=0`。
10. 状态栏和顶部导航栏间距为 `0`。
11. nav 外内容区左右边距为 `16`；卡片组间距为 `8`。
12. `tabbar` 悬浮，底部安全距离 `16dp`。
13. 类 tab 选择组只能有一个选中态。
14. 核心组件缺失时先问，不用临时 frame 冒充。
15. 规则、组件、变量、截图冲突时，在手机 Frame 外写 `设计冲突说明`。

## 生成路径

```text
触发 c2d
→ 补齐材料
→ OCR / 解读交互截图
→ 分析 visual-insight 产物（如有）
→ 读取 rules
→ 读取 MasterGo 组件库快照
→ 输出组件候选清单
→ 生成 360x780 普通 Frame 设计稿
→ 运行 hook
→ 修复 ERROR
→ 提交 MasterGo
```
