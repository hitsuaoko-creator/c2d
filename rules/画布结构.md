# 画布结构

## 推荐层级

```text
一级界面 Frame 360x780 #FFFFFF
├─ 背景层
│  └─ 一级界面顶部渐变
├─ 状态栏
├─ 顶部导航栏
├─ 内容区（flex / 自动布局）
│  ├─ top（flex / 自动布局）
│  ├─ 卡片组（flex / 自动布局）
│  └─ 键盘组件（仅键盘场景）
└─ 悬浮层
   └─ tabbar（bottom safe area = 16dp）
```

## 顶部结构

- 状态栏紧贴 Frame 顶部。
- 顶部导航栏紧贴状态栏，二者之间不能插入空白间距。
- 顶部渐变切片从 `y=0` 开始铺在背景层，覆盖状态栏和导航栏背后的视觉区域。
- 如果使用 `nav` 组件，组件上方不再额外加 spacer。

## 内容区结构

- 内容区是画板内部的 flex / 自动布局容器。
- 内容区左右边距为 `16`，nav 除外。
- `top` 用于承载导航 tab、筛选、二级 tab 或页面标题下方的控制条。
- `卡片组` 用于承载卡片、素材网格、列表、缩略图列表，gap 为 `8`。
- 键盘场景中，键盘可以位于内容区或悬浮层，但必须是 `键盘组件`。

## 悬浮层

- tabbar 属于悬浮层，不参与内容区正常流式排版。
- tabbar 底部到一级界面 Frame 底部的安全距离为 `16dp`。
- Bottom Sheet、Toast、Modal、点击标记可放在悬浮层。
- 画板外的流程箭头和说明不放进手机 Frame 的悬浮层，而放在流程板外层。

## 图层治理

- 单个 `360 x 780` 画板只做普通 Frame，不做自动布局。
- 画板内部组件必须优先放入语义化 flex / 自动布局容器，例如 `内容区`、`卡片组`、`top`。
- 背景层、状态栏、导航栏、内容区、悬浮层必须分组清楚。
- 组件实例保持为组件实例，不拆散为内部零件。
- 同一语义区域只保留一个父容器；不要让卡片、标题、图片、标签散落在页面根层。
- 标注、红框、箭头、说明文字不得进入交付界面 Frame。

## 交付前自检

- [ ] 单个设计稿画板是普通 Frame，尺寸 `360 x 780`，不是自动布局。
- [ ] 画板内部的 `内容区`、`卡片组`、`top` 等语义容器使用 flex / 自动布局。
- [ ] 一级界面 Frame 背景为 `#FFFFFF`。
- [ ] `一级界面顶部渐变` 存在，并且顶到 `y=0`。
- [ ] 状态栏与顶部导航栏间距为 `0`。
- [ ] nav 以外内容区左右边距为 `16`。
- [ ] 卡片组间距为 `8`。
- [ ] tabbar 在悬浮层，底部安全距离为 `16dp`。
- [ ] 类 tab 选择组只有一个选中态。
- [ ] 组件实例未被拆散，键盘使用 `键盘组件`。
- [ ] 标注、红框、箭头未混入交付 Frame。
