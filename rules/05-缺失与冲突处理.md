# 缺失与冲突处理

## 缺失组件策略

| 情况 | 策略 |
| --- | --- |
| 基础布局缺少对应组件 | 允许临时 frame，自由绘制容器 |
| 文本、说明、弱信息缺少组件 | 允许临时 frame 或普通文本节点 |
| 复杂业务卡片缺少匹配复合组件 | 先尝试 `封面卡片组件` / `图标卡片组件` / `点击卡片组件` / `上传卡片组件`；仍不匹配时询问 |
| 用户明确要求组件库还原，但库中没有该组件 | 停下来询问，不用素材组件冒充 |
| 需要绘制键盘 | 必须调用 keyboard/键盘 下当前导出的 `键盘组件`；如果快照中缺失该组件，停下来询问或刷新组件库 |
| 需要完整抽屉 | 当前快照没有完整 `drawer` 父组件；可用临时 frame 承载 `drawer_header` / `drawer_input`，若必须组件化则停下来询问 |
| 最外层被生成为自动布局 | 必须改为普通 Frame；内容区内部才允许自动布局 |
| 缺少 `一级界面顶部渐变` | 必须补齐切片，并放到背景层 `y=0` |
| 图标库为空或缺少图标 | 使用 `icon` 组件已有变体；仍缺失时询问，或在自由绘制场景用 FontAwesome |
| 组件 props 不确定 | 停下来询问，或选择最接近且可解释的 props |
| slot 不存在 | 不伪造 slot；改用临时 frame 或询问 |
| instance_swap 不存在 | 不声明 `instance_swap`；当前库全部为空 |

## 允许临时 frame 的边界

允许：

- 页面外层容器、分组容器、背景、分割区域。
- 辅助说明文案、非核心装饰。
- 组件库没有覆盖的布局骨架。
- 变量库没有覆盖的字号、间距、圆角、阴影。

不允许：

- 用临时 frame 替代已有 `button`、`input`、`list_switch`、`modal` 等明确组件。
- 用临时 frame 或图片素材替代 `键盘组件`。
- 用 `***素材` 组件替代业务组件。
- 用内部组件拼出已有复合组件。
- 在组件已有 props 的情况下改写或臆造 props。
- 把标注、红框、箭头或说明文字混入交付界面 Frame。
- 让 tabbar 参与内容区流式布局，或贴到屏幕底部没有 `16dp` 安全距离。

## 冲突优先级

当 design spec、组件说明、颜色文档冲突时，按以下优先级处理：

1. 明确的用户需求
2. MasterGo 组件 JSON 的真实 `name`、`props`、`slots`
3. MasterGo 变量 JSON 的真实变量名
4. 设计规范文档中的描述
5. AI 基于命名和结构的推断

## 必须生成设计稿内说明的冲突

出现以下情况时，应在设计稿中新增「设计冲突说明」文档区块：

- 组件名暗示用途，但 props 或 slot 无法支撑该用途。
- design spec 要求使用某组件，但组件库没有对应组件。
- 颜色文档要求某色值，但变量库不存在对应变量。
- 浅色和深色变量缺口不一致，导致无法一一映射。
- 用户要求使用素材，但 c2d 规则禁止素材作为页面组件入口。
- 键盘组件缺失或命名与 `键盘组件` 不一致。
- 预期稿要求普通 Frame，但生成稿最外层为自动布局。
- 预期稿要求 `一级界面顶部渐变`，但生成稿缺失或位置不是 `y=0`。
- 状态栏与顶部导航栏之间出现空隙。
- tabbar 未悬浮或底部安全距离不是 `16dp`。
- 组件说明、变量说明和实际组件 JSON 互相矛盾。

## 冲突记录模板

```md
## 设计冲突说明

- 冲突编号：C-YYYYMMDD-001
- 冲突类型：组件 / 变量 / 颜色 / 文案 / 交互
- 涉及对象：
- design spec 说明：
- 组件 JSON / 变量 JSON 事实：
- 冲突点：
- 本次处理：
- 需要确认的问题：
- 负责人 / 确认人：
```

## 当前已知冲突与限制

| 编号 | 类型 | 事实 | 影响 | 处理 |
| --- | --- | --- | --- | --- |
| C-001 | 组件说明缺失 | 40 个组件的 `description` 均为空 | 组件语义只能从命名和 props 推断 | 文档中显式标注推断，不把推断当官方说明 |
| C-002 | 预览缺失 | 40 个组件的 `cover` 均为空 | 无法视觉验证组件形态 | c2d 只使用明确 name / props / slots |
| C-003 | 图标库为空 | `icons.json` 当前为空 | 不能用 `<ui-icon>` 引用库图标 | 使用 `icon` 组件已有变体，或询问 |
| C-004 | 变量类型不完整 | 只有颜色变量，没有字号、间距、圆角、阴影变量 | 部分样式需要字面值 | 字面值只用于缺失变量类型，并遵守设计协议 |
| C-005 | 素材易误用 | 存在 `1:1 图片素材`、`3 纵列图片素材` | AI 容易把素材当业务组件 | 明确禁止图片素材作为 c2d 页面组件入口 |
| C-006 | 键盘组件规则更新 | `键盘组件` 已作为组件导出，props 为 `按键皮肤键盘 1` / `按键皮肤键盘 2` | 旧文档把键盘按素材处理会误导 c2d | 后续画键盘必须调用 `键盘组件` |
| C-007 | 抽屉父组件缺失 | 当前快照只有 `drawer_header` / `drawer_input`，没有完整 `drawer` | 直接调用 `drawer` 会失败 | 完整抽屉用临时 frame 组合局部组件，或先询问 |
| C-008 | 图层结构混乱 | 生成稿把根 Frame、自动布局、组件实例、标注层混在一起 | 图层不可读，c2d 难以维护 | 按背景层 / 状态栏 / 顶部导航栏 / 内容区 / 悬浮层重组 |
