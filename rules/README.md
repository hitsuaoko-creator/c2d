## 先读结论

1. 设计稿基准尺寸为 `360 x 780`。除 nav 以外，内容区到屏幕左右边缘的间距固定为 `16`。
2. 卡片组之间的间距为 `8`。
3. c2d 生成界面时，业务卡片区域优先调用以「组件」命名的复合组件，例如 `封面卡片组件`、`图标卡片组件`、`点击卡片组件`、`上传卡片组件`。
4. 禁止把 `1:1 图片素材`、`3 纵列图片素材` 等「素材」组件组当成页面组件直接调用；键盘已经是 keyboard/键盘 下的 `键盘组件`，画键盘时直接调用该组件。
5. `卡片_封面`、`卡片_内容区` 更接近复合卡片的内部结构，不作为 c2d 页面生成的优先入口；标签类内容优先使用 `类别标签` 或 `玩法标签（素材营销、账号权益）`。
6. `360 x 780` 是单个设计稿画板容器，画板本身必须是普通 Frame，不能是自动布局；画板内部的 `内容区`、`卡片组`、`top` 等语义容器应使用 flex / 自动布局承载组件自适应。
7. 组件不要给“固定”布局，应尽量给“适应”；除画板基准尺寸、内容边距、卡片间距等规范明确值外，不要把内部组件宽高锁死。
8. 每个类似 tab 的选择组只能有一个选中态。
9. 组件缺失或字段不确定时，允许使用临时 frame 兜底；但涉及核心业务组件、素材替代组件、或设计变量冲突时，应停下来询问并记录冲突。

## 快速规则

c2d 页面生成时按以下顺序选择：

1. 能表达完整业务块：选择 `****组件` 复合组件。
2. 表单、列表、导航、弹窗、反馈：选择对应基础组件。
3. 键盘区域：直接调用 keyboard/键盘 下的 `键盘组件`，不要用图片素材或临时 frame 代替。
4. 只作为图片资源的内容：不要直接调用 `***素材`。
5. 组件没有描述、cover 或可靠变体说明：宁可用临时 frame 复刻布局，也不要猜测未知组件。
6. design spec、组件说明、颜色变量冲突：在设计稿内新增「设计冲突说明」文档区块，并同步记录到 [冲突模板](./05-缺失与冲突处理.md#冲突记录模板)。
