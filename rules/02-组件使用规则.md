# 组件使用规则

目标：生成设计稿时先匹配组件库，不轻易自创图层；调用时只使用组件快照中真实存在的 `name`、`props`、`slots`。

## 组件选择顺序

1. 截图区域能对应完整业务块时，优先使用复合组件。
2. 复合组件不匹配时，使用基础组件或列表、弹窗、抽屉局部组件。
3. 组件库无对应组件时，才允许 `临时容器-用途`。
4. 核心组件缺失时停止询问：键盘、顶部导航、tabbar、业务卡片、弹窗、保存/确认主操作。
5. 禁止用 `***素材` 或内部结构组件冒充业务组件。

## 组合规则

### 页面基础组合

```text
360x780 普通 Frame
├─ 背景层：一级界面顶部渐变
├─ 状态栏：状态栏
├─ 顶部导航栏：导航组件
├─ 内容区：卡片、列表、上传、弹层触发区域
└─ 悬浮层：tabbar / bottom sheet / toast / 弹窗
```

一级界面必须包含 `一级界面顶部渐变`、`状态栏`、顶部导航。tabbar 是悬浮层，不进入内容区流式布局。

### 卡片组合

- 皮肤、壁纸、字体、社区封面：调用 `封面卡片组件`，不要拆 `卡片_封面` + `卡片_内容区`。
- 首页快捷入口：调用 `首页金刚区卡片组件`。
- 首页内容区：优先 `首页卡片`。
- 分类入口：调用 `图标卡片组件`。
- 表情位、素材位、可选择卡：调用 `点击卡片组件`。
- 上传和回显：调用 `上传卡片组件`。

### 键盘组合

- 任何键盘区域必须调用 `键盘组件`。
- 可用 props：`{"属性 1":"按键皮肤键盘 1"}` 或 `{"属性 1":"按键皮肤键盘 2"}`。
- 不允许用图片、素材、普通 frame 或截图复刻键盘。

### 列表和选择组合

- `list_switch` 用于 radio / checkbox / toggle 列表项。
- `list_enter` 用于点击进入型列表项。
- 单独的 `radio`、`checkbox`、`toggle` 只用于局部控件，不优先手拼完整列表项。
- tab、radio、checkbox、选项卡等同一选择组只能有一个选中态。

### 抽屉和弹层组合

- 当前库没有完整 `drawer` 父组件。
- 抽屉容器允许用 `临时容器-抽屉`，内部优先放 `抽屉顶部栏`、`drawer_input`、`option`、`按钮组组件`。
- 弹窗使用 `弹窗组件`；底部操作用 `按钮组组件` 或 `按钮组件`。
- Toast 使用 `toast`，不要自由绘制替代。

## 禁止调用入口

以下组件不得作为业务页面入口：

- `1:1 图片素材`
- `3 纵列图片素材`
- `placeholder_pic`
- `卡片_封面`
- `卡片_内容区`
- `图标卡片组件-图标`
- `withicon`

例外：用户明确要求编辑复合组件内部结构时，可以使用内部结构组件；用户明确要求整理素材库时，可以查看素材组件，但不得用它们生成业务 UI。

## props 纪律

1. `name` 必须来自 `.mastergo/library/.../components/*.json`。
2. `props` 只能使用 JSON 中存在的字段和值，不能改写、翻译或纠错。
3. 组件没有 slot 时，不伪造子节点。
4. 当前组件 `instance_swap` 均为空，不声明 `instance_swap`。
5. 组件描述为空时，只能基于名称和 props 做“推断”，不能当成官方说明。

示例：

```html
<ui-component
  data-name="keyboard"
  name="键盘组件"
  class="self-stretch"
  props='{"属性 1":"按键皮肤键盘 1"}'
></ui-component>
```

```html
<ui-component
  data-name="primary-action"
  name="按钮组件"
  props='{"主题":"主按钮","状态":"可点击","尺寸":"lg"}'
></ui-component>
```

```html
<ui-component
  data-name="cover-card"
  name="封面卡片组件"
  props='{"属性 1":"皮肤","属性 2":"价格"}'
></ui-component>
```
