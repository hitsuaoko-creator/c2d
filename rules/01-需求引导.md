# 需求引导

Use this reference when the user asks how to invoke this skill, asks for a prompt/template, or gives an incomplete design request.

## 裸触发

以下输入也必须直接触发引导流程，不需要用户先写完整需求：

```text
c2d
/c2d
设计
使用 c2d
输入法APP设计规范
输入法APP-C2D
```

历史兼容输入 `input-method-app-design` / `$input-method-app-design` 也可以触发，但对外引导优先使用 `c2d`。

如果用户通过斜杠唤起 skill 后没有输入任何正文，也按裸触发处理。先给最短模板，再按反问策略补齐缺失信息。

## 反问策略

Ask at most 3 blocking questions. Prefer this order:

1. `目标画布链接是哪个？`
2. `交互截图是否已上传，或者截图路径是什么？`
3. `组件库是否就是 https://mastergo.iflytek.com/file/196518374727340?file=196518374727340&page_id=1%3A6578？`
4. `是否已有 visual skill / visual-insight 产物？没有也可以继续。`

If the user already provided those, do not ask again. Continue with OCR, component-library reading, and generation.

## Safe Defaults

Use these assumptions unless the user says otherwise:

- Component library: `https://mastergo.iflytek.com/file/196518374727340?file=196518374727340&page_id=1%3A6578`
- Visual output: ask once; if absent, continue and mark that no visual artifact was provided
- Screen size: `360 x 780`
- First-level screen background: `#FFFFFF`
- Top gradient: `一级界面顶部渐变` at `y=0`
- Keyboard: `键盘组件`
- Content padding outside nav: `16`
- Card-group gap: `8`
- tabbar bottom safe area: `16dp`
- Flow arrows and labels stay outside phone Frames.

## Response Style

When guiding the user, keep the response short and actionable:

- Give one minimal template first.
- Then optionally provide the full template.
- Mention that missing items will be asked back as blocking questions.
- Do not paste the full design rules unless asked.
