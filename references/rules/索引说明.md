# 输入法APP 设计规范引导 rules 索引说明

本目录是 `输入法APP 设计规范引导` 仓库合并进 skill 后的内置 rules。

使用关系：

- `input-method-app-design` 是触发器和执行流程。
- `references/rules/` 是规范本体和判断依据。
- `references/硬规则.md`、`references/组件调用.md`、`references/画布结构.md` 是从 rules 中提炼出来的高频执行摘要。
- `scripts/check_mastergo_html.py` 是提交前 hook，用来拦截常见结构错误。

执行设计稿任务时：

1. 先读 `README.md`。
2. 生成设计稿前至少读：
   - `03-D2C调用准则.md`
   - `02-组件组合规则.md`
   - `06-画布结构与图层治理.md`
3. 做组件关系判断时读：
   - `00-组件与变量清单.md`
   - `01-组件关系图谱.md`
4. 做变量、颜色、冲突处理时读：
   - `04-变量与颜色规则.md`
   - `05-缺失与冲突处理.md`

如果外部仓库 `/Users/chang/Documents/Playground/输入法APP 设计规范引导` 后续更新，需要同步复制到本目录，保持 skill rules 为最新版本。
