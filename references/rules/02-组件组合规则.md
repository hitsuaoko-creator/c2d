# 组件组合规则

## 页面层级

推荐页面结构：

1. 页面 frame：`360 x 780`
2. 背景层：`一级界面顶部渐变` 顶到 `y=0`
3. 状态栏
4. 顶部导航栏：与状态栏间距 `0`
5. 内容区：复合业务组件、列表组件、表单组件；nav 以外左右边距 `16`
6. 操作区：`button` / `ButtonSpecial` / `button_group`
7. 键盘区：需要画键盘时调用 `键盘组件`
8. 悬浮层：tabbar 距离底部安全距离 `16dp`
9. 反馈层：`modal` / `toast` / `placeholder`；抽屉仅有局部组件 `drawer_header` / `drawer_input`

## 间距与自适应

- 内容区到屏幕边缘的横向间距为 `16`，nav 除外。
- 卡片组之间的间距为 `8`。
- 组件自身不要“固定”死宽高，应交给父容器和组件规范自适应；只有页面 frame、内容边距、卡片组间距这类规范值需要明确。
- 典型写法：页面根节点是 `360 x 780`，内容容器设置左右 `16`，卡片组父容器设置 `gap=8`，卡片或键盘组件实例使用适应父容器的宽度。
- 列表、卡片流、键盘等区域优先使用自适应宽度，避免写死截图里某一次选中态的 `W/H` 数值。

## 常见组合

| 组合 | 父组件 | 子/伴随组件 | 规则 |
| --- | --- | --- | --- |
| 一级到四级导航 | `nav` | `nav_Level1_tab` / `nav_Level2_tab` / `nav_Level3_tab` / `nav_Level4_tab` | 页面导航优先用 `nav`；单独标签只在局部 tab 还原时使用 |
| 封面商品卡 | `封面卡片组件` | `卡片_封面`、`卡片_内容区`、`类别标签`、`玩法标签（素材营销、账号权益）` | D2C 调用父组件，不拆用内部组件；单独标签场景使用标签组件 |
| 图标分类卡 | `图标卡片组件` | `图标卡片组件-图标` | D2C 调用父组件；内部图标组件只在局部视觉调试时使用 |
| 键盘展示 | `键盘组件` | 无 | 画键盘时直接调用，不再按素材处理 |
| 列表选择项 | `list_switch` | `radio` / `checkbox` / `toggle` / `withicon` | `theme` 决定控件类型，`icon` 决定是否展示图标块 |
| 进入型列表 | `list_enter` | 图片、描述、标记区域 | 根据 `theme` 选择是否带图、描述或标记 |
| 抽屉输入 | 临时容器 | `drawer_header` / `drawer_input` / `input` / `button_group` | 当前快照没有完整 `drawer` 父组件；需要抽屉时用临时容器承载局部组件 |
| 弹窗操作 | `modal` | `button` / `button_group` / `option` | 弹窗变体决定按钮数量和是否带选项 |
| 空状态 | `placeholder` | `placeholder_pic` | 优先调用 `placeholder`，不要直接调用图片素材 |

## 父子关系

### 卡片体系

- 父级：`封面卡片组件`
- 内部结构：`卡片_封面` + `卡片_内容区`
- 可选标签：`类别标签` / `玩法标签（素材营销、账号权益）`
- 图片资源素材：`1:1 图片素材`、`3 纵列图片素材`
- D2C 规则：生成业务卡片时只调用 `封面卡片组件`、`图标卡片组件`、`点击卡片组件`、`上传卡片组件` 等复合组件。

### 键盘体系

- 父级：keyboard/键盘 下当前导出的 `键盘组件`
- 变体：`按键皮肤键盘 1` / `按键皮肤键盘 2`
- D2C 规则：后续往界面画键盘时，直接调用 `键盘组件`；不要使用图片素材、普通 frame 或旧的素材规则替代。

### 列表体系

- 父级：`list_switch` 或 `list_enter`
- 控制节点：`radio`、`checkbox`、`toggle`
- 辅助节点：`withicon`
- D2C 规则：生成列表项时调用 `list_switch` / `list_enter`，不要手动拼 `radio` + 文案 + 分割线，除非该列表样式在组件库中缺失。

### 抽屉体系

- 当前快照没有完整 `drawer` 父组件。
- 头部：`drawer_header`
- 输入：`drawer_input`
- 输入 slot：`drawer_input` 的 `input`
- 操作：`button_group` 或 `button`
- D2C 规则：需要抽屉时，允许使用临时 frame 作为抽屉容器，但头部和输入区域优先调用 `drawer_header` / `drawer_input`；若用户要求完整抽屉组件，先停下来确认或刷新组件库。

### 导航体系

- 父级：`nav`
- 子 tab：`nav_Level1_tab` 到 `nav_Level4_tab`
- D2C 规则：完整导航优先调用 `nav`，只在页面内小范围 tab 时调用 level tab。

### 弹窗与反馈体系

- 弹窗：`modal`
- 操作：`button_group` / `button`
- 多选项内容：`option`
- Toast：`toast`
- 空状态：`placeholder`
- D2C 规则：反馈组件应直接调用库组件，不用自由绘制替代。

## 变体选择准则

1. 状态只来自组件 props：例如 `checked` / `unchecked`、`on` / `off`、`selected` / `unselected`。
2. 不改写 props 名称和枚举值：例如 `title_description.` 末尾句点必须保留。
3. 英文拼写错误也按组件原文使用：例如 `without descripiton`、`withdecripiton`、`enter_withdescription_withdescripition`。
4. 没有 slot 的组件不要伪造子节点；没有 `instance_swap` 的组件不要伪造替换关系。
5. 类 tab 的选择组只能有一个选中项；同一组内不能同时出现多个 `checked`、`selected` 或 `选中` 状态。
