# D2C 调用准则

## 组件选择优先级

1. 页面基准尺寸为 `360 x 780`。
2. 除 nav 以外，内容区到屏幕左右边缘的间距为 `16`。
3. 卡片组之间的间距为 `8`。
4. 业务卡片：优先选 `****组件` 命名的复合组件。
5. 键盘区域：直接调用 keyboard/键盘 下当前导出的 `键盘组件`。
6. 基础交互：选 `button`、`input`、`radio`、`checkbox`、`toggle` 等基础组件。
7. 页面容器和反馈：选 `nav`、`modal`、`toast`、`placeholder`；当前快照没有完整 `drawer` 父组件。
8. 资源素材：不要直接选 `***素材`。
9. 规范说明或命名不明组件：没有明确用途前不要调用。

## 布局硬规则

- `360 x 780` 是单个设计稿画板容器尺寸，不是内容区尺寸。
- 单个设计稿画板必须是普通 Frame，不能是自动布局。
- 画板内部组件和语义容器应使用 flex / 自动布局组织，例如 `内容区`、`卡片组`、`top`、列表、网格行等；不要把所有子节点散落在画板根层。
- 一级界面背景色为 `#FFFFFF`。
- 一级界面必须包含 `一级界面顶部渐变`，且该切片顶到 Frame 最上方 `y=0`。
- 状态栏和顶部导航栏间距为 `0`。
- nav 可以贴边或遵循自身组件规范；nav 以外的内容容器左右边距为 `16`。
- 卡片组之间使用 `8` 间距。
- tabbar 是悬浮层，距离底部安全距离 `16dp`。
- 组件实例尽量使用自适应布局：画板内部的 flex 容器负责 `16` 边距和 `8` 卡片组间距；组件自身宽度优先 `self-stretch` 或由父容器控制，高度优先随内容和组件自身规范适应。
- 除页面基准、内容边距、卡片组间距、明确的组件规范尺寸外，不要给组件内部写死固定宽高。
- 类 tab 的选择组只能有一个选中态；同一组内只能存在一个 `checked` / `selected` / `选中`，其余必须为 `unchecked` / `unselected` / `未选中`。

## 图层治理硬规则

- 根 Frame 下按 `背景层` / `状态栏` / `顶部导航栏` / `内容区` / `悬浮层` 分组。
- 标注、红框、箭头、说明文字不能混入交付界面 Frame。
- 组件实例不要拆散成零件；临时 frame 只用于组件库缺失的容器骨架。
- 同一语义区域只保留一个父容器，避免卡片标题、图片、标签散落在根层。
- 更完整的层级和自检见 [画布结构与图层治理](./06-画布结构与图层治理.md)。

## 必须调用的组件入口

业务卡片和内容块：

- `封面卡片组件`
- `图标卡片组件`
- `点击卡片组件`
- `上传卡片组件`
- `键盘组件`

列表和表单：

- `list_switch`
- `list_enter`
- `input`
- `drawer_input`
- `option`

弹层和反馈：

- `modal`
- `toast`
- `placeholder`

操作：

- `button`
- `ButtonSpecial`
- `button_group`

导航：

- `nav`
- `nav_Level1_tab`
- `nav_Level2_tab`
- `nav_Level3_tab`
- `nav_Level4_tab`

## 禁止直接调用的组件入口

除非用户明确要求还原素材本身，否则不要在 D2C 页面生成中调用：

- `1:1 图片素材`
- `3 纵列图片素材`
- `placeholder_pic`

除非正在编辑复合组件内部结构，否则不要优先调用：

- `卡片_封面`
- `卡片_内容区`
- `图标卡片组件-图标`
- `withicon`

## HTML 调用示例

示例只展示命名和 props 结构，实际页面需按 MasterGo 逆向转译协议补齐布局类和 `data-name`。

```html
<ui-component
  data-name="cover-card"
  name="封面卡片组件"
  props='{"属性 1":"皮肤","属性 2":"价格"}'
></ui-component>
```

```html
<ui-component
  data-name="switch-list-item"
  name="list_switch"
  props='{"theme":"toggle","status":"checked","info":"with descripiton","icon":"withicon"}'
></ui-component>
```

```html
<ui-component
  data-name="primary-action"
  name="button"
  props='{"主题":"主按钮","状态":"可点击","尺寸":"lg"}'
></ui-component>
```

```html
<ui-component
  data-name="keyboard"
  name="键盘组件"
  class="self-stretch"
  props='{"属性 1":"按键皮肤键盘 1"}'
></ui-component>
```

## 不合规示例

不要用素材组生成业务卡片：

```html
<ui-component data-name="bad-card" name="1:1 图片素材" props='{"属性 1":"1"}'></ui-component>
```

不要把内部结构当作完整业务组件：

```html
<ui-component data-name="bad-card-content" name="卡片_内容区" props='{"属性 1":"通用_带标签"}'></ui-component>
```

## 当前库限制

- 组件 `description` 为空，无法从官方说明确认用途。
- 组件 `cover` 为空，无法通过预览图确认视觉。
- `icons.json` 为空，不能使用 `<ui-icon>` 引用库图标。
- 当前快照没有完整 `drawer` 父组件；只有 `drawer_header` 和 `drawer_input`。
- 所有 `instance_swap` 为空，不能声明内部替换通道。

因此，D2C 生成时只能使用组件已有 `name`、`props`、`slots`。未知内容用临时 frame 或询问处理。
