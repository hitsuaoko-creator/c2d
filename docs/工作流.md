# 工作流

## c2d Generation Flow

1. Trigger `/c2d`, `c2d`, or `设计`.
2. Enter guided intake if materials are missing.
3. Collect target MasterGo canvas, interaction screenshot, component library, and visual-insight output if available.
4. OCR and summarize the interaction screenshot.
5. Summarize visual-insight output into `本次视觉生成概要`.
6. Read `rules/` and MasterGo component-library snapshot.
7. Build component shortlist.
8. Generate MasterGo HTML.
9. Run `scripts/check_mastergo_html.py`.
10. Submit to MasterGo only after hook errors are fixed.

## Required Inputs

- Target MasterGo canvas URL.
- Interaction screenshot or screenshot path.
- Component library, default `https://mastergo.iflytek.com/goto/TOAnEBPS`.
- visual-insight output when available.

## Output Before Submission

- Screenshot/OCR summary.
- Visual generation summary.
- Component shortlist.
- Temporary containers and reasons.
- Hook result.
