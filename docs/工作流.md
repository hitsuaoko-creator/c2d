# 工作流

## c2d 生成流程

1. 通过 `/c2d`、`c2d` 或 `设计` 触发。
2. 如果材料不完整，先进入引导式信息收集。
3. 收集目标 MasterGo 画布、交互截图、组件库，以及可选的 visual-insight 产物。
4. 对交互截图做 OCR 和结构解读。
5. 将 visual-insight 产物总结为 `本次视觉生成概要`。
6. 读取 `rules/` 和 MasterGo 组件库快照。
7. 建立组件候选清单。
8. 生成 MasterGo HTML。
9. 运行 `scripts/check_mastergo_html.py`。
10. 只有 hook 错误修复后，才提交到 MasterGo。

## 必要输入

- 目标 MasterGo 画布链接。
- 交互截图或截图路径。
- 组件库，默认 `https://mastergo.iflytek.com/goto/TOAnEBPS`。
- visual-insight 产物，有则提供，没有可继续。

## 提交前输出

- 交互截图/OCR 解读。
- `本次视觉生成概要`。
- 组件候选清单。
- 临时容器及原因。
- hook 结果。
