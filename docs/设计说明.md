# 设计说明

| 模块 | 说明 |
| --- | --- |
| 背景 | **避免只做截图复刻**：输入法 APP 的 UI 生成不能停留在“看见什么就画什么”，否则容易遗漏状态反馈、失败态、空态、误触、闭环、移动端适配、信息层级、转化动作等关键设计判断。<br><br>**避免组件误用**：MasterGo 组件库中同时存在业务组件、基础组件、内部结构组件和素材组件。如果不建立调用规则，AI 容易把 `***素材` 当成业务组件，或把组件实例拆散成无语义图层。<br><br>**避免多工具割裂**：同一套设计规范需要在 Codex、Cursor、Claude Code 中都能用 `/c2d` 唤起，不能只服务某一个 IDE。 |
| 目标 | 建立一个跨 Agent 的 `c2d` Skill，让 AI 在生成输入法 APP 设计稿前，先读取交互截图、visual skill 产物、MasterGo 组件库和输入法设计 rules，再沉淀成本次生成策略，最终输出符合组件规范、画布规范和视觉意图的高保真 MasterGo 设计稿。 |
| Skill 逻辑 | AI 通过 `/c2d` 或 `c2d / 设计 / 使用 c2d` 进入引导式流程。先补齐目标 MasterGo 画布、交互截图、组件库和 visual 产物；再对交互截图做 OCR/视觉解读；如果提供 visual skill 产物，则先总结为 `本次视觉生成概要`；随后读取 `rules/` 和 MasterGo 组件库，建立截图区域与组件库组件的映射；最后生成设计稿并运行 hook 校验。 |
| Skill 结构 | 以 `rules/` 为规范源，承载输入法 APP 组件关系、组合规则、D2C 调用准则、变量颜色、缺失冲突和画布治理。以 `SKILL.md` 作为 Codex/Cursor/Claude Skill 入口，以 `AGENTS.md` 作为通用 Agent 入口，以 `CLAUDE.md` 作为 Claude Code 项目入口，以 `.claude/commands/c2d.md` 和 `.cursor/commands/c2d.md` 支持项目级 slash command，以 `docs/` 和 `prompts/` 承载人类说明和可复制 prompt。 |
| 工作流程 | `触发 /c2d` → `引导补齐材料` → `OCR 交互截图` → `分析 visual 产物并生成 本次视觉生成概要` → `读取 rules` → `读取 MasterGo 组件库` → `建立组件候选清单` → `生成 360x780 Frame 设计稿` → `运行 scripts/check_mastergo_html.py` → `修复 ERROR 后提交 MasterGo`。 |
| 产物 | 1. 可被 Codex、Cursor、Claude Code 识别的 `c2d` Skill。<br>2. 可通过 `/c2d` 唤起的 Claude/Cursor command。<br>3. 拉平后的 `rules/` 规范源。<br>4. `视觉产物解读规则.md`，用于把 visual skill 产物转成设计生成概要。<br>5. `scripts/install_c2d.sh` 和 `scripts/install_slash_commands.sh`，用于安装三端 skill 与项目级 slash command。<br>6. 生成前必须输出的 `交互截图解读`、`本次视觉生成概要`、`组件候选清单`、`hook 结果`。 |

## 结构

```text
c2d/
├── AGENTS.md
├── CLAUDE.md
├── SKILL.md
├── .claude/commands/c2d.md
├── .cursor/commands/c2d.md
├── agents/openai.yaml
├── docs/
├── prompts/
├── rules/
└── scripts/
```

## 不会做的

- 不会跳过 OCR/截图解读直接画 UI。
- 不会跳过 visual 产物概要直接套组件。
- 不会把 `1:1 图片素材`、`3 纵列图片素材`、`placeholder_pic` 当业务组件使用。
- 不会把 `360 x 780` 画板做成自动布局。
- 不会把组件实例拆散成无语义图层。
- 不会在 hook 存在 `ERROR` 时提交到 MasterGo。
