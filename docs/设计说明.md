# 3. 设计说明

| 模块 | 说明 |
| --- | --- |
| 背景 | **避免主观输出**：只看截图容易把设计讨论停在“好不好看”，很难解释为什么某处要突出、某处要弱化，也容易忽略路径、状态、CTA、信息层级和转化动作。<br><br>**避免人工遗漏**：输入法 APP 的设计稿包含状态栏、顶部导航、一级界面顶部渐变、tabbar、键盘、底部弹层、素材位、选中态、回填态等固定结构。如果没有规则约束，AI 容易漏掉失败态、空态、误触、闭环、移动端适配、图层命名和组件父子关系。<br><br>**避免组件误用**：MasterGo 组件库中同时存在业务组件、基础组件、内部结构组件和素材组件。没有调用准则时，AI 容易把 `***素材` 当成业务组件，或把组件实例拆散成无语义图层。<br><br>**避免多工具割裂**：同一套规范需要在 Codex、Cursor、Claude Code 中都能通过 `/c2d` 或“设计”触发，不能只服务某一个 IDE。 |
| 目标 | 将输入法 APP 设计生成中最容易散、漏、主观化的部分结构化，沉淀一套跨 Agent 可复用的 `c2d` Skill。AI 在生成 UI 前先读取交互截图、visual skill 产物、MasterGo 组件库和 `rules/` 规则，建立截图意图、视觉策略、组件映射和硬规则之间的证据关系，最终输出可维护、可检查、可继续迭代的高保真 MasterGo 设计稿。 |
| Skill 逻辑 | AI 引导式收集目标画布、交互截图、组件库、visual 产物等信息；先做 OCR/视觉解读，再把 visual skill 产物总结为 `本次视觉生成概要`；随后读取 `rules/` 和 MasterGo 组件库，判断哪些区域可直接调用组件、哪些区域需要组合组件、哪些区域只能临时 frame；最后生成设计稿，运行 hook 校验，并在提交前输出截图解读、组件候选清单、临时容器原因和冲突说明。<br><br><pre><code class="language-mermaid">flowchart LR<br>  A[输入需求/截图/画布/组件库/visual产物] --> B[判断材料完整度]<br>  B -- 缺失 --> C[引导补齐信息]<br>  B -- 完整 --> D[OCR 交互截图]<br>  C --> D<br>  D --> E[总结 visual 产物]<br>  E --> F[读取 rules 规则]<br>  F --> G[读取 MasterGo 组件库]<br>  G --> H[建立组件候选清单]<br>  H --> I[生成设计稿]<br>  I --> J[运行 hook 校验]<br>  J --> K[提交 MasterGo]</code></pre> |
| Skill 结构 | 以 `rules/` 为输入法 APP 设计规范源，约束组件关系、组件组合、c2d 调用、变量颜色、缺失冲突、画布结构和图层治理。以 `SKILL.md` 作为 Codex/Cursor/Claude Skill 入口，以 `AGENTS.md` 作为通用 Agent 入口，以 `CLAUDE.md` 作为 Claude Code 项目入口，以 `.claude/commands/c2d.md` 和 `.cursor/commands/c2d.md` 支持项目级 slash command，以 `docs/` 和 `prompts/` 承载人类说明和可复制 prompt。<br><br><details><summary>结构</summary><br><br><pre><code>c2d/<br>├── AGENTS.md                      # 通用 Agent 执行入口<br>├── CLAUDE.md                      # Claude Code 项目入口<br>├── SKILL.md                       # Skill 主入口，触发 rules<br>├── .claude/commands/c2d.md        # Claude Code slash command<br>├── .cursor/commands/c2d.md        # Cursor slash command<br>├── agents/openai.yaml             # Codex Skill 列表展示信息<br>├── docs/                          # 人类说明文档<br>├── prompts/                       # 可复制主 Prompt<br>├── rules/                         # 输入法APP 设计规范引导 rules<br>└── scripts/                       # 安装脚本与提交前 hook</code></pre><br></details> |
| 工作流程 | 触发方式支持 `/c2d`、`c2d`、`设计`、`使用 c2d`，也支持斜杠唤起后不输入内容直接发送。进入后不直接画图，而是先补齐输入材料，再做截图解读、visual 概要、rules 读取、组件库读取、组件候选清单、HTML 生成、hook 校验和 MasterGo 提交。<br><br><details><summary>prompt（已写入 skill，无需人工重复输入）</summary><br><br><pre><code>/c2d<br><br>目标画布：&lt;MasterGo 画布链接&gt;<br>组件库：https://mastergo.iflytek.com/goto/TOAnEBPS<br>交互截图：&lt;上传截图或截图路径&gt;<br>visual 产物：&lt;粘贴 visual skill 输出；没有则写暂无&gt;<br>任务类型：生成 / 优化 / 审核<br>补充约束：&lt;页面数量、链路、必须保留的组件或例外规则&gt;</code></pre><br></details><br><br><details><summary>知识库（已写入 skill，生成前必须读取）</summary><br><br>- `rules/需求引导.md`：材料缺失时如何引导用户补齐。<br>- `rules/硬规则.md`：360x780 Frame、内容区 16、卡片组 8、一级界面顶部渐变、tabbar、键盘等死规则。<br>- `rules/组件调用.md`：组件优先级、禁止调用素材组件、临时 frame 策略。<br>- `rules/画布结构.md`：画板和内部 flex 的边界。<br>- `rules/视觉产物解读规则.md`：如何把 visual skill 产物总结成生成概要。<br>- `rules/02-组件组合规则.md`、`rules/03-c2d调用准则.md`、`rules/06-画布结构与图层治理.md`：组件组合、c2d 和图层治理。<br></details> |
| 产物 | 1. 可被 Codex、Cursor、Claude Code 识别的 `c2d` Skill。<br>2. 可通过 `/c2d` 唤起的 Claude/Cursor command。<br>3. 拉平后的 `rules/` 规范源，作为“输入法APP 设计规范引导”的规则层。<br>4. `视觉产物解读规则.md`，用于把 visual skill 产物转成设计生成概要。<br>5. `scripts/install_c2d.sh` 和 `scripts/install_slash_commands.sh`，用于安装三端 skill 与项目级 slash command。<br>6. 生成前必须输出的 `交互截图解读`、`本次视觉生成概要`、`组件候选清单`、`hook 结果`。<br><br><details><summary>不会做的</summary><br><br>- 不会跳过 OCR/截图解读直接画 UI。<br>- 不会跳过 visual 产物概要直接套组件。<br>- 不会把 `1:1 图片素材`、`3 纵列图片素材`、`placeholder_pic` 当业务组件使用。<br>- 不会把 `360 x 780` 画板做成自动布局。<br>- 不会把组件实例拆散成无语义图层。<br>- 不会在 hook 存在 `ERROR` 时提交到 MasterGo。<br></details> |
